## Код для исследования
```java

public class JvmComprehension {

    public static void main(String[] args) {
        int i = 1;                      // 1
        Object o = new Object();        // 2
        Integer ii = 2;                 // 3
        printAll(o, i, ii);             // 4
        System.out.println("finished"); // 7
    }

    private static void printAll(Object o, int i, Integer ii) {
        Integer uselessVar = 700;                   // 5
        System.out.println(o.toString() + i + ii);  // 6
    }
}

// 1 Первоначально происходит иницилизация класса JvmComprehension и информация о нем, а также его полях и методах 
// загружаются в Metaspace. В момент вызова метода main для него создается фрейм в стеке. В этом фрейме произойдет
// создание локальной переменной примитивного типа int и присвоится значение 1.

// 2 В куче создается объект класса Object,после вызова конструктора new Object(),а в фрейме метода main создается 
// ссылка на этот объект, которая сохраняется в локальной переменной o.

// 3 В куче создается объект класса Integer со значением 2,а в фрейме метода main создается ссылка на этот объект, 
// которая сохраняется в локальной переменной ii.

// 4 Вызывается метод printAll с тремя аргументами o, i и ii. Это приводит к созданию нового фрейма метода printAll в
// стеке. Ссылки на o и ii, а также копия примитивного значения i передаются в новый фрейм метода printAll.

// 5 Создается новый объект Integer со значением 700 в куче. В стеке во фрейме метода printAll создается новая локальная
// переменная uselessVar, в которой сохранется ссылка на этот объект.

// 6 Создается фрейм метода println, в котором хранится результат сложения o.toString(), i и ii. При этом на вершине
// стека создается еще один фрейм для метода toString, в котором хранится ссылка на уже созданный объект o. После 
// завершения этих двух методов соответствующие фреймы стека удаляются, а управление возвращается к коду, 
// который вызвал этот метод.

// 7 Создается фрейм метода println и выводится строка finished. При этом сама строка хранится в пуле строк в куче.

// Код класса JvmComprehension загружается в JVM с помощью Application Classloader.

// Сборка мусора происходит периодически. Garbage Collector собирает объекты из памяти, которые больше не используются 
// (не доступны для использования). В данном примере это может произойти после выполнения метода printAll, когда
// переменная uselessVar и аргументы o, i и ii становятся недоступными. После завершения метода main все его переменные
// также станут недоступными для использования.
```
